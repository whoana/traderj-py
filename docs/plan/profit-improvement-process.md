# 수익률 개선 프로세스 (초안)

> **상태:** 초안 — 회의 후 설계서로 발전 예정
> **작성일:** 2026-03-25
> **목적:** 백테스트 결과를 기반으로 레짐/전략/파라미터를 체계적으로 개선하는 반복 프로세스 정의

---

## 1. 문제 인식

현재 수익률 개선 작업이 **산발적이고 비체계적**이다.

```
현재 상태 (끊어진 파이프라인):

  백테스트 실행 → 결과 확인 → ... → (수동으로 코드 수정) → 재배포
                              ↑
                          여기서 끊김
                          - 분석은 보여주지만 적용 불가
                          - 최적화 결과를 저장/재사용 못함
                          - 개선 효과 검증 방법 없음
```

### 구체적 문제들

| # | 문제 | 현재 상태 |
|---|------|----------|
| 1 | 최적화 결과 적용 불가 | Optuna 결과를 보여주기만 함. "적용" 버튼 없음 |
| 2 | 검증 루프 없음 | 최적화 파라미터로 재백테스트 불가 |
| 3 | 프리셋 고정 | presets.py 하드코딩. 런타임 변경 시 재시작하면 초기화 |
| 4 | 레짐 매핑 검증 어려움 | 매핑 변경 후 실제 개선 확인 방법 없음 |
| 5 | 이력 관리 없음 | 어떤 변경이 어떤 결과를 냈는지 추적 불가 |
| 6 | 전략 전환 시 포지션 충돌 | 전략 바뀌어도 기존 포지션이 잠금 → 새 전략 진입 불가 |

## 2. 목표 프로세스

```
┌─────────────────────────────────────────────────────────┐
│              수익률 개선 사이클 (PDCA)                     │
│                                                         │
│   ① 백테스트    →  ② 분석    →  ③ 최적화   →  ④ 적용    │
│       ↑                                          │      │
│       └──────────── ⑤ 검증 ←─────────────────────┘      │
│                       │                                  │
│                  통과 시                                   │
│                       ↓                                  │
│                  ⑥ 배포 (페이퍼/라이브)                    │
└─────────────────────────────────────────────────────────┘
```

### ① 백테스트 실행

- 기간, 전략, 모드 선택하여 실행
- 모드: 단일전략 / 전략비교 / AI레짐 / 최적화
- **입력:** 기간 + 전략 (or 전체)
- **출력:** 수익률, 승률, MDD, 거래수 등 메트릭

### ② 분석

- 레짐별 성과 분해 (어떤 레짐에서 어떤 전략이 좋았나)
- 전략 간 비교 (최고/최저 성과 전략)
- 시장 대비 알파 확인
- **입력:** 백테스트 결과
- **출력:** 인사이트 + 액션 제안 (전략전환 / 레짐매핑 / 파라미터최적화)

### ③ 최적화 (3가지 경로)

```
분석 결과에 따라 선택:

  A. 전략 전환
     → 최고 성과 전략으로 즉시 전환
     → 가장 단순, 즉시 효과

  B. 레짐 매핑 변경
     → 레짐별 최적 전략 재배치
     → 레짐-전략 매핑 테이블 업데이트
     → 예: RANGING_HIGH_VOL → STR-003 대신 STR-006

  C. 파라미터 최적화
     → Optuna로 Tier 1 파라미터 탐색 (threshold, weights)
     → 현재 프리셋 ±30% 범위에서 최적값 탐색
     → Top 3 후보 도출
```

### ④ 적용 ← 현재 누락된 핵심 단계

```
최적화 결과를 엔진에 반영하는 3가지 방법:

  a. 임시 프리셋 생성 (검증용)
     → 최적 파라미터로 런타임 프리셋 생성
     → STRATEGY_PRESETS에 등록 (메모리 내)
     → 검증 백테스트에서 사용 가능

  b. 영구 프리셋 저장 (확정 후)
     → 검증 통과한 프리셋을 DB에 저장
     → 또는 presets.py에 새 프리셋으로 추가

  c. 기존 프리셋 업데이트
     → 기존 STR-XXX의 파라미터 직접 수정
     → 이력 관리 필요 (before/after)
```

### ⑤ 검증

- 적용된 파라미터로 **동일 기간** 재백테스트
- baseline(변경 전) vs optimized(변경 후) 비교
- **통과 기준 (예시):**
  - 수익률 개선 ≥ 0.5%p
  - MDD 악화 ≤ 1%p
  - 거래수 ≥ 5 (과적합 방지)
- 미통과 시 → ③으로 돌아가 재최적화

### ⑥ 배포

- 검증 통과 시:
  - 페이퍼 트레이딩에 적용 (1~2주 관찰)
  - 페이퍼 결과 양호 시 → 라이브 반영 (미래)
- 변경 이력 기록 (날짜, 변경내용, 사유, 결과)

## 3. 현재 구현 vs 필요 구현

| 단계 | 현재 | 필요 | 우선순위 |
|------|------|------|---------|
| ① 백테스트 | ✅ 4개 모드 (single/compare/ai_regime/optimize) | - | - |
| ② 분석 | ✅ analyzer.py + ActionPanel UI | - | - |
| ③-A 전략전환 | ✅ switch 버튼 동작 | - | - |
| ③-B 레짐매핑 | ✅ apply-regime-map API | - | - |
| ③-C 파라미터최적화 | ✅ Optuna 30 trials | - | - |
| ④ 적용 | ❌ 없음 | 임시 프리셋 생성 + 등록 | P0 |
| ⑤ 검증 | ❌ 없음 | 적용 후 자동 재백테스트 + 비교 | P0 |
| ⑥ 배포 | 🔶 수동 (코드 수정 + fly deploy) | hot-swap + 이력 관리 | P1 |
| 이력 관리 | ❌ 없음 | 변경 이력 DB 테이블 | P1 |
| 포지션 충돌 해결 | ❌ 미해결 | 전환 시 포지션 처리 정책 | P2 |

## 4. 열린 질문 (회의에서 결정 필요)

### Q1: 최적화 프리셋 저장 방식?
- **A안:** DB 테이블에 저장 (런타임 로드)
- **B안:** presets.py에 코드로 추가 (배포 필요)
- **C안:** JSON/YAML 설정 파일 (코드 변경 없이 반영)

### Q2: 검증 통과 기준은?
- 수익률 개선 최소 몇 %?
- MDD 악화 허용 범위?
- 최소 거래 수? (과적합 방지)
- 복수 기간 교차 검증 필요?

### Q3: 레짐 매핑 변경 범위?
- 현재 4개 레짐 × 4개 전략 (고정)
- 6개 레짐 (bull/bear 포함) 매핑을 AI가 자동 제안?
- 매핑 변경도 검증 루프 필요?

### Q4: 전략 전환 시 포지션 처리?
- 방안 1: 전환 시 기존 포지션 자동 청산 (깔끔하지만 강제 손실 가능)
- 방안 2: 전략별 독립 포지션 (자금 분산)
- 방안 3: SL/TP만 새 전략 기준으로 재조정 (절충)

### Q5: 자동화 수준?
- 완전 수동: 사용자가 각 단계를 클릭
- 반자동: "최적화 → 검증" 자동, 적용은 수동 승인
- 완전 자동: 주기적으로 자동 실행 + 자동 적용 (AI Tuner 최종 목표)

### Q6: Tier 2/3 파라미터도 포함?
- 현재 최적화는 Tier 1(시그널)만 대상
- Tier 2(리스크: SL/TP/trailing) 포함 시 검색 공간 폭발
- 단계적 확장? (먼저 Tier 1 안정화 → Tier 2 추가)

## 5. 대시보드 UX 흐름 (구상)

```
백테스트 페이지
  │
  ├─ 실행: 기간/전략 선택 → Run
  │
  ├─ 결과: 차트 + 메트릭 테이블
  │
  ├─ 히스토리: 과거 백테스트 목록
  │     └─ 클릭 → 결과 상세
  │
  └─ 분석 & 액션 패널 (ActionPanel)
        │
        ├─ [전략 전환] → 즉시 반영
        │
        ├─ [레짐 매핑 적용] → 즉시 반영
        │
        └─ [파라미터 최적화] → Optuna 실행
              │
              └─ 결과 표시 (baseline vs 최적화)
                    │
                    ├─ [검증 백테스트] ← 신규: 최적 파라미터로 재실행
                    │     └─ 비교 결과 (before/after)
                    │
                    └─ [적용] ← 신규: 엔진에 반영
                          └─ 확인 다이얼로그 → hot-swap
```
