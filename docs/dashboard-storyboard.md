# TraderJ Dashboard — UI/UX 스토리보드

**작성일:** 2026-03-22
**목적:** Next.js 모니터링 대시보드 개발 전 사전 검토
**플랫폼:** Web (Next.js App Router, Vercel 배포)
**데이터 소스:** TraderJ Engine API (`https://traderj-engine.fly.dev/api/v1/*`)

---

## 1. 디자인 시스템

### 1.1 스타일 방향

| 항목 | 결정 | 근거 |
|------|------|------|
| 테마 | Dark Mode Primary | 금융 트레이딩 대시보드 표준. 장시간 모니터링 시 눈 피로 감소 |
| 스타일 | Accessible & Data-Dense | 실시간 수치 중심, 높은 가독성, WCAG AA 준수 |
| 정보 밀도 | High-density compact | 한 화면에 핵심 지표 최대 노출 (트레이더 관행) |
| 반응형 | Desktop-first, Mobile 지원 | 주 사용 환경이 데스크톱이나 모바일 알림 확인 필요 |

### 1.2 컬러 팔레트 (Financial Dashboard Dark)

| 토큰 | 색상 | 용도 |
|------|------|------|
| `--bg` | `#020617` (slate-950) | 전체 배경 |
| `--surface` | `#0F172A` (slate-900) | 카드/패널 배경 |
| `--surface-raised` | `#1E293B` (slate-800) | 호버/활성 카드 |
| `--border` | `#334155` (slate-700) | 구분선, 카드 테두리 |
| `--text-primary` | `#F8FAFC` (slate-50) | 주 텍스트 |
| `--text-secondary` | `#94A3B8` (slate-400) | 보조 텍스트, 레이블 |
| `--text-muted` | `#64748B` (slate-500) | 비활성 텍스트 |
| `--positive` | `#22C55E` (green-500) | 수익, 매수, 상승 |
| `--negative` | `#EF4444` (red-500) | 손실, 매도, 하락 |
| `--accent` | `#3B82F6` (blue-500) | 주요 액션, 링크 |
| `--warning` | `#F59E0B` (amber-500) | 경고 상태 |
| `--destructive` | `#DC2626` (red-600) | 위험 액션 (긴급정지 등) |

### 1.3 타이포그래피

| 용도 | 폰트 | 크기 | Weight |
|------|------|------|--------|
| 숫자/데이터 | Fira Code | 14-32px | 500-700 |
| UI 텍스트 | Fira Sans (또는 Inter) | 12-18px | 400-600 |
| 라벨 | Fira Sans | 11-12px | 500 (uppercase) |

> 숫자에 Tabular Figures(고정폭 숫자) 적용 — 가격 변동 시 레이아웃 시프트 방지

### 1.4 차트 라이브러리

| 차트 유형 | 라이브러리 | 용도 |
|-----------|-----------|------|
| 캔들스틱 (OHLCV) | Lightweight Charts (TradingView) | 가격 차트 |
| 라인/에어리어 | Recharts 또는 Chart.js | PnL 곡선, 에쿼티 커브 |
| 게이지/불릿 | Custom SVG 또는 ApexCharts | KPI 달성률 |
| 히트맵 | Custom SVG | 시간대별 거래 분포 |

### 1.5 아이콘 & 컴포넌트

- 아이콘: **Lucide React** (일관된 stroke width, dark mode 최적)
- UI 라이브러리: **shadcn/ui** (Tailwind CSS 기반, 커스터마이즈 용이)
- 애니메이션: **Framer Motion** (상태 전환, 진입/퇴장)

## 2. 사이트맵 & 네비게이션

```
TraderJ Dashboard
│
├── / (Dashboard)              ← 메인 화면, 핵심 지표 한눈에
│
├── /positions                 ← 포지션 상세
│   └── /positions/history     ← 과거 포지션
│
├── /orders                    ← 주문 이력
│
├── /analytics                 ← 분석
│   ├── /analytics/pnl         ← PnL 곡선
│   ├── /analytics/signals     ← 시그널 이력
│   └── /analytics/compare     ← 전략 비교
│
├── /control                   ← 엔진 제어
│   ├── 엔진 시작/정지/재시작
│   ├── 전략 변경
│   └── SL/TP 수동 조정
│
└── /settings                  ← 설정 확인
    ├── 현재 전략 설정
    ├── 레짐 상태
    └── 시스템 정보
```

### 네비게이션 패턴

- **Desktop**: 좌측 고정 사이드바 (아이콘 + 텍스트, 축소 가능)
- **Mobile**: 하단 탭 바 (5개 이하: Dashboard, Positions, Analytics, Control, Settings)
- 상단 헤더: 엔진 상태 인디케이터 + 긴급 정지 버튼 + 알림

**인터랙션:**

- 종료된 포지션: 페이지네이션, 정렬 (날짜/PnL)
- 포지션 행 클릭 → 상세 확장 (진입 시그널, 주문 ID, 슬리피지)

- [정지] → 확인 다이얼로그("엔진을 정지합니까? 트레이딩이 중단됩니다") → `POST /engine/stop`
- [재시작] → 확인 → `POST /engine/restart`
- 전략 라디오 선택 + [전략 변경 적용] → 확인 → `POST /strategy/switch`
- 모든 제어 액션은 성공/실패 토스트 표시

**위험 액션 UX 규칙:**
- 빨간색 버튼 + 확인 다이얼로그 + 3초 카운트다운 (긴급정지)
- 전략 변경: 현재 포지션 유무에 따른 경고 메시지 분기

## 6. 기술 구현 계획

### 6.1 기술 스택

| 항목 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | Next.js 14+ (App Router) | SSR, API Routes 프록시 |
| 배포 | Vercel | 글로벌 Edge, 무료 티어 |
| 스타일링 | Tailwind CSS + shadcn/ui | 컴포넌트 커스터마이즈 |
| 차트 | Lightweight Charts + Recharts | 캔들스틱 + 일반 차트 |
| 상태관리 | TanStack Query (React Query) | API 캐싱, 폴링 |
| 아이콘 | Lucide React | 일관된 아이콘셋 |
| 모노레포 | dashboard/ 디렉토리 | 기존 프로젝트 구조 유지 |

### 6.2 API 프록시 패턴

```
[사용자 브라우저]
  ↓ (쿠키/세션)
[Vercel Next.js API Routes]
  ↓ (X-API-Key, 서버사이드)
[Fly.io TraderJ Engine API]
```

- API 키는 Vercel 환경변수에만 저장 → 브라우저 노출 없음
- Next.js API Routes가 프록시 역할
- CORS 문제 없음 (같은 도메인)

### 6.3 데이터 갱신 전략

| 데이터 | 방식 | 주기 | 라이브러리 |
|--------|------|------|-----------|
| 엔진 상태 | 폴링 | 30초 | TanStack Query refetchInterval |
| 잔고/PnL | 폴링 | 30초 | TanStack Query |
| 캔들 차트 | 폴링 | 60초 | TanStack Query + Lightweight Charts |
| 포지션 | 폴링 | 15초 | TanStack Query |
| 매크로 | 폴링 | 5분 | TanStack Query |
| 설정/버전 | 1회 | 페이지 로드 | TanStack Query staleTime: Infinity |
| 제어 응답 | 즉시 | POST 후 refetch | TanStack Query invalidation |

### 6.4 반응형 브레이크포인트

| 브레이크포인트 | 크기 | 레이아웃 |
|---------------|------|---------|
| Mobile | < 768px | 하단 탭, 1열, 축소된 KPI |
| Tablet | 768-1023px | 사이드바 축소, 2열 그리드 |
| Desktop | 1024-1439px | 사이드바 확장, 3열 그리드 |
| Wide | ≥ 1440px | 풀 사이드바, 4열 그리드, 넓은 차트 |
