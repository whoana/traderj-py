# AI 전략 자동 튜닝 설명서

**TraderJ의 AI Tuner가 어떻게 전략을 스스로 개선하는지 쉽게 설명합니다.**

---

## 한 줄 요약

> 매주 AI가 지난 거래 성적을 분석하고, 더 나은 설정값을 찾아서 적용한 뒤, 48시간 동안 지켜보다가 성적이 나빠지면 자동으로 원래대로 되돌립니다.

---

## 1. 왜 필요한가?

트레이딩 봇은 "언제 사고 팔지"를 **숫자(파라미터)** 로 결정합니다.

예를 들어:
- **매수 기준점** = 0.10 → "점수가 0.10 이상이면 산다"
- **매도 기준점** = -0.08 → "점수가 -0.08 이하면 판다"
- **손절 배율** = 2.0 → "변동성의 2배 아래로 떨어지면 손절한다"

문제는 시장이 계속 변한다는 것입니다. 상승장에서 잘 먹히던 설정이 횡보장에서는 안 통할 수 있습니다. 지금까지는 사람이 직접 숫자를 바꿔야 했는데, AI Tuner가 이걸 자동으로 합니다.

---

## 2. 어떻게 동작하나? (4단계)

```
평가 → 최적화 → 적용 → 감시
```

### 1단계: 평가 (지난주 성적표 만들기)

지난 7일간의 거래 기록을 보고 성적표를 만듭니다:
- 승률이 얼마나 됐나?
- 수익은 손실의 몇 배인가? (수익 팩터)
- 최대 얼마나 떨어졌나? (최대 낙폭)
- 시그널 정확도는?

→ **"지난주에 이 전략이 얼마나 잘했는지"** 를 숫자로 정리합니다.

### 2단계: 최적화 (더 나은 설정 찾기)

두 가지 방법을 **동시에** 사용합니다:

| 방법 | 비유 | 하는 일 |
|------|------|---------|
| **통계 엔진 (Optuna)** | 수학 선생님 | 50가지 설정 조합을 시뮬레이션해서 가장 수익이 좋은 3개를 골라냄 |
| **AI (Claude)** | 경험 많은 트레이더 | 성적표를 보고 "이 시장에서는 이렇게 바꾸면 좋겠다"고 의견을 냄 |

→ 통계 엔진이 후보를 만들고, AI가 그 중 가장 적합한 걸 선택합니다.

### 3단계: 적용 (설정 변경)

선택된 새 설정값을 실시간 봇에 적용합니다. 이때 **안전장치(가드레일)** 가 작동합니다:

- 한 번에 **최대 20%** 까지만 변경 가능 (급격한 변화 방지)
- 설정값이 허용 범위를 벗어나면 자동으로 범위 내로 조정
- 특정 비중값들의 합이 1.0이 되도록 자동 정규화

### 4단계: 감시 (48시간 모니터링)

새 설정으로 바꾼 뒤 48시간 동안 주의 깊게 지켜봅니다:

- **최대 낙폭이 기준의 2배를 넘으면** → 자동 롤백 (원래 설정으로 복구)
- **연속 5회 손실** → 자동 롤백
- **48시간 무사 통과** → 새 설정 확정!

## 3. 3단계 파라미터 등급

모든 설정이 같은 중요도는 아닙니다. 위험도에 따라 3등급으로 나뉩니다:

### Tier 1 — 시그널 파라미터 (매주 자동)

"언제 사고 팔지"를 결정하는 가장 기본적인 설정입니다.

| 파라미터 | 쉬운 설명 | 범위 |
|----------|-----------|------|
| 매수 기준점 | 이 점수 이상이면 "사자" | 0.03 ~ 0.25 |
| 매도 기준점 | 이 점수 이하면 "팔자" | -0.20 ~ -0.02 |
| 1시간봉 비중 | 단기 차트를 얼마나 참고할지 | 10% ~ 80% |
| 4시간봉 비중 | 중기 차트를 얼마나 참고할지 | 10% ~ 80% |
| 일봉 비중 | 장기 차트를 얼마나 참고할지 | 0% ~ 80% |
| 매크로 비중 | 공포탐욕지수 등 외부 지표 반영도 | 0% ~ 30% |

→ 비교적 안전한 설정이라 **매주 월요일 자동으로** 최적화됩니다.

### Tier 2 — 리스크 파라미터 (격주 자동)

"얼마나 사고, 언제 손절할지"를 결정합니다. 돈에 직접 영향을 미칩니다.

| 파라미터 | 쉬운 설명 | 범위 |
|----------|-----------|------|
| 손절 배율 | 변동성 대비 손절 거리 | 1.5x ~ 3.0x |
| 보상/위험 비율 | 이익 목표 대 손절 거리 비율 | 1.5 ~ 4.0 |
| 추적 손절 활성화 | 수익이 이만큼 나면 추적 손절 시작 | 0.5% ~ 3% |
| 최대 포지션 크기 | 자산 중 최대 투자 비율 | 5% ~ 30% |
| 일일 최대 손실 | 하루에 잃을 수 있는 최대치 | 3% ~ 8% |

→ 돈과 직접 관련되므로 **격주(2주마다)** 변경하며, 같은 방향으로 연속 변경 불가합니다.
(예: 포지션 크기를 지난번에 올렸으면, 이번에는 올릴 수 없음)

### Tier 3 — 레짐 파라미터 (월간, 사람 승인 필요)

"시장 상태를 어떻게 판단할지"를 결정합니다. 전략 전체의 성격을 바꿀 수 있어 가장 신중합니다.

| 파라미터 | 쉬운 설명 | 범위 |
|----------|-----------|------|
| 추세 판단 기준 | ADX가 이 값 이상이면 "추세장" | 20 ~ 35 |
| 변동성 판단 기준 | 볼린저밴드 폭이 이 값 이상이면 "고변동" | 2% ~ 8% |
| 확인 횟수 | 시장 상태 변경 전 연속 확인 필요 횟수 | 2 ~ 5회 |
| 쿨다운 시간 | 전략 전환 후 최소 대기 시간 | 30 ~ 120분 |

→ **매월 1회** 평가하며, **반드시 사람이 승인해야** 적용됩니다.

## 4. 안전장치 정리

AI가 멋대로 전략을 망치지 않도록 여러 겹의 안전장치가 있습니다:

```
[변경 전]
  ├─ 최대 변경폭 제한 (±20%)
  ├─ 절대 범위 제한 (허용 범위 초과 불가)
  ├─ Tier 2 동일 방향 연속 변경 차단
  └─ Tier 3 사람 승인 필수

[변경 후]
  ├─ 48시간 실시간 모니터링
  ├─ 최대 낙폭 초과 시 자동 롤백
  ├─ 연속 5회 손실 시 자동 롤백
  └─ 연속 3회 롤백 시 튜너 일시 중지 (사람 개입 필요)
```

## 5. AI 프로바이더 체계

AI 분석은 외부 서비스를 사용하며, 장애에 대비한 3중 구조입니다:

```
1순위: Claude (Anthropic)     ← 메인 AI
  ↓ 장애 시
2순위: GPT-4o (OpenAI)        ← 백업 AI
  ↓ 장애 시
3순위: Degraded Mode          ← AI 없이 통계만으로 결정
```

- 각 AI 서비스에 **회로 차단기** 적용: 3회 연속 실패 시 10분간 차단 후 재시도
- **월 예산 $5 제한**: 초과 시 자동으로 Degraded Mode 전환
- Degraded Mode에서도 Optuna 통계 엔진은 정상 동작하므로 기본적인 최적화는 계속됩니다

## 6. 자동 스케줄

| 주기 | 동작 | Tier |
|------|------|------|
| 매주 월요일 00:00 (UTC) | 시그널 파라미터 최적화 | Tier 1 |
| 격주 월요일 00:00 (UTC) | + 리스크 파라미터 최적화 | Tier 1 + 2 |
| 매월 첫째 주 월요일 | + 레짐 파라미터 최적화 (승인 대기) | Tier 1 + 2 + 3 |
| 매시간 | 모니터링 중인 세션 점검 (롤백/확정 판단) | - |

---

## 7. 대시보드/API로 할 수 있는 것

| 동작 | API | 설명 |
|------|-----|------|
| 튜닝 이력 보기 | `GET /tuning/history` | 과거 튜닝 결과 확인 |
| 튜너 상태 확인 | `GET /tuning/status` | 현재 튜너 상태, 모니터링 세션 |
| AI 상태 확인 | `GET /tuning/provider-status` | Claude/OpenAI 장애 여부, 잔여 예산 |
| 수동 튜닝 실행 | `POST /tuning/trigger/{전략}` | 스케줄을 기다리지 않고 바로 실행 |
| 수동 롤백 | `POST /tuning/rollback/{ID}` | 마음에 안 드는 변경 즉시 되돌리기 |
| Tier 3 승인 | `POST /tuning/approve/{ID}` | 레짐 파라미터 변경 승인/거부 |

## 8. 실제 동작 예시

```
월요일 00:00  튜너 가동
              ↓
  [평가] STR-001의 지난 7일: 승률 40%, 수익팩터 1.2, 낙폭 3%
              ↓
  [최적화] Optuna가 50가지 설정 시뮬레이션
           → 상위 3개 후보 추출
           Claude에게 "이 시장에서 어떤 후보가 좋을까?" 질의
           → Claude: "2번 후보 추천, 매수 기준점을 0.12로 올리면 좋겠다"
              ↓
  [가드레일] 변경폭 확인: 0.10 → 0.12 (20% 변경) ✓ 허용 범위 내
              ↓
  [적용] 실시간 봇에 새 설정 반영
         Telegram 알림: "STR-001 튜닝 적용 — buy_threshold 0.10→0.12"
              ↓
  [감시] 48시간 모니터링 시작
              ↓
  수요일 00:00  48시간 경과, 낙폭 정상 → 확정!
                Telegram 알림: "STR-001 튜닝 확정"
```

만약 화요일에 갑자기 큰 손실이 발생하면:

```
  화요일 15:00  [감시] 낙폭 6% 감지 (기준 3%의 2배 초과!)
                → 자동 롤백: buy_threshold 0.12 → 0.10 (원래 값)
                Telegram 알림: "STR-001 튜닝 롤백 — MDD 초과"
```
