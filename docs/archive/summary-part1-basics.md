# 파이썬으로 배우는 알고리즘 트레이딩 - Part 1: 기초편 (Ch.00~09)

## 학습 개요

이 교재는 파이썬 프로그래밍 언어를 사용하여 주식 데이터를 효율적으로 분석하고, 자신만의 투자 기법을 평가하며, 궁극적으로 자동화된 트레이딩 시스템을 구축하는 것을 목표로 한다. 기초편(Ch.00~09)에서는 파이썬의 기본 문법부터 클래스, 파일 처리, COM을 활용한 엑셀 제어까지 알고리즘 트레이딩에 필요한 프로그래밍 기초 역량을 단계적으로 쌓아간다. 프로그래밍을 모르는 주식 투자자와 프로그래밍은 알지만 주식 투자 경험이 없는 개발자 모두를 대상으로 한다.

## Ch.01 파이썬 시작하기

## Ch.02 파이썬 변수와 문자열

### 핵심 내용
- **계산**: 파이썬을 계산기처럼 사용 (사칙연산, 나머지, 거듭제곱)
- **변수**: 데이터를 저장하는 공간. 파이썬에서 변수는 객체를 참조(바인딩)하는 이름
- **문자열**: 텍스트 데이터 처리
  - **인덱싱**: `문자열[인덱스]`로 특정 위치 문자 접근. 음수 인덱스는 뒤에서부터 역순
  - **슬라이싱**: `문자열[시작:끝]`으로 부분 문자열 추출. 시작/끝 생략 가능
  - **자르기**: `split()` 메서드로 문자열을 구분자 기준으로 분리
  - **합치기**: `+` 연산자 또는 `join()` 메서드로 문자열 결합
- **데이터 타입**: 정수(int), 실수(float), 문자열(str), 불리언(bool)

### 주요 코드 패턴
```python
# 변수와 계산
price = 50000
quantity = 10
total = price * quantity

# 문자열 슬라이싱
code = "005930"  # 삼성전자 종목코드
market = code[:3]  # 앞 3자리

# 문자열 분리/결합
data = "삼성전자,005930,50000"
items = data.split(",")  # ['삼성전자', '005930', '50000']
```

### 트레이딩 연관성
- 종목코드, 종목명 등의 문자열 처리는 주식 데이터 다루기의 기본
- 가격, 수량 등의 수치 계산은 매매 금액 산출에 필수
- API에서 받아오는 데이터는 대부분 문자열이므로 형변환 및 파싱 능력 필요

## Ch.03 파이썬 기본 자료 구조

### 핵심 내용
- **리스트(list)**: 순서가 있는 변경 가능한 데이터 집합
  - 생성, 인덱싱, 슬라이싱, 데이터 삽입(`append`, `insert`), 삭제(`del`, `remove`)
- **튜플(tuple)**: 순서가 있는 변경 불가능한 데이터 집합
  - 리스트와 유사하나 한번 생성하면 수정 불가
  - 슬라이싱 가능
- **딕셔너리(dict)**: 키(key)와 값(value)의 쌍으로 데이터를 저장
  - 데이터 삽입/삭제, 키 목록 구하기(`keys()`), 값 목록 구하기(`values()`)

### 주요 코드 패턴
```python
# 리스트 - 종목 리스트 관리
stocks = ["삼성전자", "SK하이닉스", "LG화학"]
stocks.append("카카오")

# 튜플 - 변경되면 안 되는 데이터
market_hours = (9, 0, 15, 30)  # 개장시간, 폐장시간

# 딕셔너리 - 종목 정보 관리
stock_info = {"code": "005930", "name": "삼성전자", "price": 50000}
print(stock_info["name"])  # 삼성전자
```

### 트레이딩 연관성
- 리스트: 관심종목 리스트, 매수/매도 대기열 관리
- 딕셔너리: 종목별 정보(코드, 이름, 가격, 거래량 등)를 체계적으로 저장
- 증권사 API에서 반환하는 데이터를 자료구조에 담아 관리하는 것이 기본

## Ch.04 제어문

### 핵심 내용
- **Boolean과 논리연산자**: True/False, and, or, not
- **if문**: 조건에 따른 분기 처리 (if, elif, else)
- **for문**: 정해진 횟수만큼 반복 (range, 리스트/튜플/딕셔너리 순회)
- **while문**: 조건이 참인 동안 반복. 반복 횟수가 정해지지 않은 경우 사용
- **break와 continue**: 반복문 제어
- **중첩 루프**: 반복문 안에 반복문

### 주요 코드 패턴
```python
# if문 - 매매 조건 판단
current_price = 49000
target_price = 50000
if current_price < target_price:
    print("매수")
elif current_price > target_price * 1.1:
    print("매도")
else:
    print("관망")

# for문 - 종목 리스트 순회
for stock in ["삼성전자", "SK하이닉스", "카카오"]:
    print(f"{stock} 데이터 조회 중...")

# while문 - 시장 감시
import time
market_open = True
while market_open:
    # 주가 체크 로직
    # 조건 충족시 break
    break
```

### 트레이딩 연관성
- **if문**: 매수/매도 조건 판단 (목표가 도달, 손절가 도달 등)이 알고리즘 트레이딩의 핵심
- **for문**: 전 종목 순회하며 조건 검색, 과거 데이터 분석
- **while문**: 장 시간 동안 지속적으로 시장을 모니터링하는 루프 구현
- for문은 반복 횟수가 정해진 경우, while문은 조건 기반 반복에 사용

## Ch.05 파이썬 함수와 모듈

### 핵심 내용
- **함수**: `def` 키워드로 정의. 코드 재사용과 구조화의 기본 단위
  - 매개변수(parameter)와 인자(argument)
  - 기본값 매개변수, 키워드 인자
- **반환값**: `return`으로 결과 반환. 여러 값 반환 가능(튜플)
- **모듈**: 파이썬 코드를 파일 단위로 구성. `import`로 불러오기

### 트레이딩 연관성
- **함수**: 매수/매도 로직, 수익률 계산, 데이터 조회 등을 함수로 모듈화
- **시간 모듈**: 장 시작/종료 시간 확인, API 호출 간격 제어(`sleep`), 데이터 타임스탬프 처리
- **OS 모듈**: 로그 파일 경로 관리, 데이터 파일 저장 디렉토리 관리
- 트레이딩 시스템은 여러 함수와 모듈의 조합으로 구성됨

## Ch.06 파이썬 클래스

### 핵심 내용
- **클래스란**: 관련된 데이터(속성)와 기능(메서드)을 하나로 묶는 설계도
- **생성자(`__init__`)**: 인스턴스 생성 시 자동 호출되는 메서드. 초기값 설정에 사용
- **self**: 인스턴스 자신을 가리키는 참조. 메서드 호출 시 파이썬이 자동으로 전달
- **네임스페이스**: 변수와 객체의 바인딩 관계를 저장하는 공간
  - 클래스가 정의되면 독립적인 네임스페이스 생성
  - 클래스 내 변수/메서드는 딕셔너리 형태로 저장
- **클래스 변수 vs 인스턴스 변수**:
  - 클래스 변수: 모든 인스턴스가 공유
  - 인스턴스 변수: 각 인스턴스마다 독립적 (`self.변수명`)
- **상속**: 기존 클래스의 기능을 물려받아 확장. `class 자식(부모):`

### 주요 코드 패턴
```python
class Stock:
    market = "KOSPI"  # 클래스 변수

    def __init__(self, code, name, price):
        self.code = code        # 인스턴스 변수
        self.name = name
        self.price = price

    def get_info(self):
        return f"{self.name}({self.code}): {self.price}원"

# 상속
class FavoriteStock(Stock):
    def __init__(self, code, name, price, target):
        super().__init__(code, name, price)
        self.target_price = target

    def should_buy(self):
        return self.price < self.target_price

samsung = FavoriteStock("005930", "삼성전자", 50000, 55000)
print(samsung.should_buy())  # True
```

### 트레이딩 연관성
- **클래스 설계**: 종목(Stock), 주문(Order), 포트폴리오(Portfolio) 등을 클래스로 모델링
- **상속**: 기본 전략 클래스를 상속하여 다양한 매매 전략 구현
- **증권사 API 연동 시 클래스 활용**: API 래퍼 클래스를 만들어 복잡한 API 호출을 캡슐화
- 실전 트레이딩 시스템의 핵심 구조가 클래스 기반

## Ch.07 파일 다루기

### 핵심 내용
- **파일 읽기**: `open()` 함수로 파일 열기, `read()`, `readline()`, `readlines()`로 읽기
  - 텍스트 파일을 이용한 주식 매수 리스트 관리 예제
- **파일 쓰기**: `open(파일명, 'w')` 또는 `'a'`(추가) 모드로 데이터 기록
- **with 문**: 파일 자동 닫기 보장

### 주요 코드 패턴
```python
# 파일 읽기 - 매수 종목 리스트 불러오기
with open("buy_list.txt", "r", encoding="utf-8") as f:
    stocks = f.readlines()
    for stock in stocks:
        print(stock.strip())

# 파일 쓰기 - 매매 기록 저장
with open("trade_log.txt", "w", encoding="utf-8") as f:
    f.write("2024-01-15, 삼성전자, 매수, 50000, 10주\n")
    f.write("2024-01-16, 삼성전자, 매도, 52000, 10주\n")
```

### 트레이딩 연관성
- **매매 로그 기록**: 거래 내역을 파일로 저장하여 추후 분석
- **설정 파일**: 매매 조건, 종목 리스트 등을 외부 파일로 관리
- **데이터 저장**: API에서 받아온 주가 데이터를 파일로 저장하여 백테스팅에 활용

## Ch.09 파이썬과 COM

### 핵심 내용
- **COM(Component Object Model)**: Windows에서 프로그램 간 통신을 위한 기술
  - 파이썬에서 COM 객체를 사용하면 엑셀, 워드 등 Windows 프로그램을 제어 가능
- **win32com.client**: COM 객체 생성을 위한 파이썬 모듈
  - `Dispatch()` 메서드로 COM 객체 생성
- **엑셀 다루기**:
  - **엑셀 파일 읽기**: Workbook과 Worksheet의 관계 이해, 셀 데이터 접근
  - **셀에 컬러 입히기**: 셀의 배경색, 글자색 등 서식 변경

### 주요 코드 패턴
```python
import win32com.client

# 엑셀 COM 객체 생성
excel = win32com.client.Dispatch("Excel.Application")
excel.Visible = True

# 워크북 열기
wb = excel.Workbooks.Open(r"C:\data\stock_data.xlsx")
ws = wb.Worksheets("Sheet1")

# 셀 읽기
stock_name = ws.Cells(1, 1).Value
stock_price = ws.Cells(1, 2).Value

# 셀에 컬러 입히기 (상한가: 빨강, 하한가: 파랑)
ws.Cells(1, 1).Interior.ColorIndex = 3  # 빨강

wb.Save()
excel.Quit()
```

### 트레이딩 연관성
- **COM은 증권사 API의 핵심 기술**: 대신증권 CybosPlus, 키움증권 OpenAPI 등 국내 증권사 API가 COM 기반
- **win32com.client.Dispatch()**: 증권사 API 객체 생성에 동일한 패턴 사용
- **엑셀 연동**: 주가 데이터를 엑셀로 내보내기, 매매 보고서 자동 생성
- COM 기술의 이해가 이후 챕터(대신증권 API, 키움증권 API)의 필수 선행 지식

## 기초편 핵심 정리

### 1. 기초 문법에서 트레이딩까지의 연결고리
| 파이썬 기초 | 트레이딩 활용 |
|---|---|
| 변수, 데이터 타입 | 종목코드, 가격, 수량 저장 |
| 문자열 처리 | API 응답 데이터 파싱 |
| 리스트/딕셔너리 | 종목 리스트, 포트폴리오 관리 |
| 조건문 (if) | 매수/매도 조건 판단 |
| 반복문 (for/while) | 종목 순회, 실시간 모니터링 |
| 함수 | 매매 로직 모듈화 |
| 클래스 | 종목, 주문, 전략 객체 설계 |
| 파일 I/O | 매매 로그, 설정 파일 관리 |
| COM (win32com) | 증권사 API 연동, 엑셀 자동화 |

### 2. 핵심 설계 패턴 (기초편에서 배운 것)
- **데이터 모델링**: 클래스로 종목, 주문 등의 도메인 객체를 설계
- **이벤트 루프**: while 루프 + 조건문으로 시장 감시 및 자동 매매 구현
- **모듈화**: 기능별로 함수와 모듈을 분리하여 유지보수성 확보
- **데이터 영속성**: 파일 I/O로 설정 및 기록 데이터를 저장/복원

### 3. 이후 학습을 위한 준비사항
- **Windows 환경**: 증권사 API가 Windows COM 기반이므로 Windows OS 필수
- **32비트 파이썬**: 증권사 API 호환을 위해 32비트 버전 아나콘다 설치
- **PyCharm IDE**: 실전 개발을 위한 통합 개발 환경 준비
- **COM 이해**: win32com.client.Dispatch() 패턴이 증권사 API 연동의 기본

### 4. 기초편 학습 완료 후 다음 단계
기초편을 마치면 Ch.10(대신증권 API), Ch.12(키움증권 API) 등 실제 증권사 API를 활용한 데이터 조회 및 자동 주문 구현으로 진행한다. 기초편에서 배운 클래스, 파일 I/O, COM 기술이 증권사 API 연동의 직접적인 기반이 된다.
