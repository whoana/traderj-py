## 2. 레짐 → 전략 매핑

```
시장 상황 (레짐)              자동 선택 전략
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
추세 + 고변동성 (TRENDING_HIGH_VOL)
  → STR-002 (Aggressive Trend Follow)
     공격적 추세추종, 큰 움직임 포착

추세 + 저변동성 (TRENDING_LOW_VOL)
  → STR-001 (Conservative Trend Follow) ← 현재 실행 중
     보수적 추세추종, 안정적 수익

횡보 + 고변동성 (RANGING_HIGH_VOL)
  → STR-003 (Hybrid Reversal)
     역추세, 박스권 상하단 매매

횡보 + 저변동성 (RANGING_LOW_VOL)
  → STR-005 (Low-Frequency Conservative)
     저빈도, 확실한 기회만 매매
```

## 3. 전략 전환 시 포지션 처리 (핵심!)

### 현재 구현: "파라미터만 교체, 포지션은 유지"

```
레짐 전환 발생!
    │
    ▼
┌────────────────────────────────────────────────┐
│ signal_generator.apply_preset(new_preset)       │
│                                                │
│ 바뀌는 것:                                      │
│  • 시그널 점수 계산 방식 (가중치, 임계값)        │
│  • 타임프레임 비중 (1h/4h/1d)                   │
│  • 매수/매도 임계값 (threshold)                  │
│                                                │
│ 바뀌지 않는 것:                                  │
│  • 기존 오픈 포지션 (그대로 유지!)               │
│  • SL/TP 설정 (기존 값 유지)                    │
│  • 잔고 (변동 없음)                             │
└────────────────────────────────────────────────┘
```

### 시나리오별 동작

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시나리오 A: STR-001(포지션 있음) → STR-002로 전환

  기존 포지션: 0.018 BTC (STR-001에서 매수)
      │
      ▼
  전략이 STR-002로 바뀜 (파라미터만 교체)
      │
      ▼
  다음 Tick에서 STR-002 기준으로 시그널 생성
      │
      ├─ BUY 시그널 → ❌ 스킵 (이미 포지션 있음!)
      ├─ HOLD 시그널 → 대기
      └─ SELL 시그널 → ✅ 매도 (STR-002 기준으로 청산)

  ⚠️ 문제: STR-001에서 산 포지션을 STR-002 기준으로 청산
     → 매도 타이밍이 원래 전략과 다를 수 있음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시나리오 B: STR-001(포지션 있음) → STR-003(역추세)로 전환

  기존 포지션: 0.018 BTC (추세추종으로 매수)
      │
      ▼
  전략이 STR-003(역추세)로 바뀜
      │
      ▼
  ⚠️ 역추세 전략은 상승 시 매도 시그널 → 빨리 청산될 수 있음
  ⚠️ 또는 추세추종에서 산 포지션이 역추세 관점에서 불리할 수 있음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시나리오 C: STR-001(포지션 없음) → STR-005로 전환

  포지션 없음
      │
      ▼
  전략이 STR-005로 바뀜
      │
      ▼
  다음 BUY 시그널부터 STR-005 기준으로 매수
  → ✅ 가장 깔끔한 케이스

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 4. 포지션 생명주기 (현재 구현)

```
                    ┌─────────┐
                    │ 포지션   │
                    │ 없음     │
                    └────┬────┘
                         │
                    BUY 시그널 + 리스크 통과
                         │
                         ▼
                    ┌─────────┐
                    │ 포지션   │ ← 전략당 1개만 허용
                    │ 보유     │
                    └────┬────┘
                         │
              ┌──────────┼──────────┬──────────┐
              │          │          │          │
         SELL 시그널   SL 발동    TP 발동   Trailing
              │          │          │       Stop 발동
              ▼          ▼          ▼          ▼
         ┌────────────────────────────────────────┐
         │            포지션 청산                   │
         │  → 실현손익 기록                        │
         │  → 다음 BUY 시그널 대기                 │
         └────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

포지션 사이징 (매수 시):

  총잔고 = KRW + (BTC × 현재가)
  가용금액 = 총잔고 - 기존포지션가치

  변동성(ATR)에 따라:
  ┌─────────────────────────────────────────┐
  │ ATR 높음 (시장 격렬) → 5% 투입 (최소)   │
  │ ATR 보통            → 6~15% 투입       │
  │ ATR 낮음 (시장 조용) → 20% 투입 (최대)  │
  └─────────────────────────────────────────┘

  투입금액 = 가용금액 × position_pct
  매수수량 = 투입금액 ÷ 현재가
```

## 5. 레짐 전환 안전장치

```
┌─────────────────────────────────────────────────┐
│ 디바운스 (debounce_count = 3)                    │
│                                                 │
│ Tick 1: 횡보 감지 → 1/3 (대기)                   │
│ Tick 2: 횡보 감지 → 2/3 (대기)                   │
│ Tick 3: 횡보 감지 → 3/3 → 전환 승인!             │
│                                                 │
│ → 일시적 노이즈로 전략이 바뀌는 것 방지           │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│ 쿨다운 (cooldown_minutes = 60)                   │
│                                                 │
│ 전환 후 60분간 추가 전환 차단                     │
│ → 전략이 빈번하게 왔다갔다 하는 것(whipsaw) 방지  │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│ 수동 잠금 (lock/unlock)                          │
│                                                 │
│ lock() → 자동 전환 완전 차단                     │
│ unlock() → 자동 전환 다시 허용                   │
│ → 수동으로 특정 전략 고정 가능                    │
└─────────────────────────────────────────────────┘
```

## 6. 현재 구조의 한계와 질문에 대한 답변

### Q: 레짐이 바뀌면 새 전략으로 매수 가능한가?

**답: 기존 포지션이 있으면 불가능.**

```
STR-001에서 매수한 포지션 보유 중
        │
   레짐 변경 → STR-005로 전환
        │
   STR-005 기준 BUY 시그널 발생
        │
   has_open_position("STR-001") = True
        │
   ❌ "Skipping buy: already_has_position"
```

모든 전략이 같은 `strategy_id`(STR-001)로 실행되므로,
전략 프리셋이 바뀌어도 `strategy_id`는 동일 → 기존 포지션이 잠금.

### Q: 전략 전환인데 기존 포지션 때문에 매수 못하면 의미 없지 않나?

**맞는 지적입니다.** 현재 구현의 한계:

```
문제 상황:
  STR-001(추세)로 매수 → 횡보장 진입 → STR-003(역추세)로 전환
  → 하지만 STR-001 포지션이 있어서 STR-003으로 신규 매수 불가
  → STR-003 기준 SELL 시그널이 나올 때까지 대기
  → 결국 "추세 전략으로 산 포지션을 역추세 기준으로 파는" 상황
```

### 개선 방안 (미구현)

```
방안 1: 전략 전환 시 기존 포지션 자동 청산
  레짐 전환 → 기존 포지션 시장가 매도 → 새 전략으로 시작
  장점: 깔끔한 전환
  단점: 손실 중인 포지션도 강제 청산

방안 2: 전략별 독립 포지션
  STR-001과 STR-003이 각각 독립 포지션 보유
  장점: 전략 간 간섭 없음
  단점: 자금 분산, 복잡도 증가

방안 3: 전환 시 SL/TP만 재조정
  포지션은 유지하되, 새 전략 기준으로 SL/TP 재계산
  장점: 포지션 유지하면서 리스크 관리 적응
  단점: 근본적 해결 아님
```

## 7. 현재 실행 중인 설정 요약

| 항목 | 값 |
|------|-----|
| 실행 전략 | STR-001 (Conservative Trend Follow) |
| 레짐 자동 전환 | 활성화 |
| 디바운스 | 3회 연속 감지 필요 |
| 쿨다운 | 60분 |
| 포지션 제한 | 전략당 1개 |
| 포지션 사이징 | 변동성 기반 5~20% |
| SL | ATR×2 또는 -3% 고정 |
| TP | R:R 2.0 (리스크의 2배) |
| Trailing Stop | +1% 수익 시 활성화, 1.5% 거리 |
